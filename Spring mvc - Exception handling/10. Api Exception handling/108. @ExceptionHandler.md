ExceptionHandlerExceptionResolver handle @ExceptionHandler
- recommended for api response 

@ExceptionHandler
- select Exception(include sub class) to handle
  - sub class have priority
- apply to same controller method
- can select multiple Exception
```java
@ExceptionHandler({AException.class, BException.class})
public String ex(Exception e){}
``` 

- can omit Exception type by using parameter type
```java
@ExceptionHandler
public String ex(UserException e){}
```


[handler process flow]
```java
@ResponseBody
@ResponseStatus(HttpStatus.BAD_REQUEST)
@ExceptionHandler(IllegalArgumentException.class)
public ErrorResult illegalExHandle(IllegalArgumentException ex){
    log.info("IllegalArgumentException handler");
    return new ErrorResult("BAD_REQUEST", ex.getMessage());
}
```
- controller throw `IllegalArgumentException`
- ExceptionResolver act. use first ExceptionHandlerExceptionResolver(high priority)
- ExceptionHandlerExceptionResolver find @ExceptionHandler for `IllegalArgumentException`
- run illegalExHandle()
- @ResponseBody, @ResponseStatus set response
- return ErrorResult in body to client
